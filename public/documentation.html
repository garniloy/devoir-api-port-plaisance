Conventions

Request format

  • All request bodies must be JSON with Content-Type: application/json
  • Path parameters are documented on each endpoint
  • Dates should be sent as ISO 8601 strings (e.g. 2024-07-15)
  • Email addresses in URLs should be percent-encoded (%40 for @)

Response format

  • All responses are JSON
  • Successful creates return 201, reads and updates return 200
  • Errors include an issue or message field and a details or error field
  • Timestamps (createdAt, updatedAt) are always UTC ISO 8601

Resource identifiers

  • Catways are identified by catwayNumber (integer), not _id
  • Reservations are identified by their MongoDB _id (ObjectId)
  • Users are identified by email address

Data storage

  • All data is persisted in MongoDB via Mongoose
  • User passwords are hashed with bcrypt (cost 10) and never returned
  • Reservations are nested under catways — always scope requests with a valid :id
