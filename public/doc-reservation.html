Reservations

Manage berth bookings nested under a catway — create, list, retrieve, update, and cancel reservations with built-in date conflict detection.

Base URL /catways/:id/reservations
ℹ️
Reservations are nested under catways. Every request requires a valid :id (the catwayNumber) as a path parameter. See the Catways documentation to understand the parent resource.

Data Model

Each reservation is stored as a MongoDB document linked to its parent catway via catwayNumber.

Reservation MongoDB / Mongoose
FieldTypeConstraintsDescription
_id ObjectId auto MongoDB unique identifier, generated automatically. Used as :idReservation in routes.
catwayNumber Number required ref Foreign reference to the parent Catway.catwayNumber. Injected automatically from the URL — do not send in the body.
clientName String required Full name of the client making the reservation.
boatName String required Name or identifier of the vessel being berthed.
startDate Date required Start of the reservation period. ISO 8601 date string is recommended.
endDate Date required End of the reservation period. Must be after startDate.
createdAt Date auto Timestamp of document creation, managed by Mongoose.
updatedAt Date auto Timestamp of last modification, managed by Mongoose.

Validation Rules

  • The :id path parameter must match an existing catwayNumber. If the catway doesn't exist, the request returns 404 before any reservation logic runs.
  • catwayNumber is set server-side from the URL — never send it in the request body. If sent, it will be ignored or may cause unexpected behaviour on updates.
  • All four body fields (clientName, boatName, startDate, endDate) are mandatory on creation.
  • Date conflict check — the API rejects any reservation whose period overlaps with an existing one on the same catway. Two reservations conflict if startDate ≤ existingEnd AND endDate ≥ existingStart.
  • Dates should be sent as ISO 8601 strings (e.g. "2024-07-15" or "2024-07-15T10:00:00.000Z"). Invalid date strings cause a Mongoose cast error and a 500 response.
  • On update, all body fields are accepted and spread directly onto the document. You are responsible for ensuring updated dates do not create new conflicts — the conflict check only runs on creation.
  • To find a specific reservation, the API matches both _id (as :idReservation) and catwayNumber. A reservation that exists but belongs to a different catway will return 404.

Endpoints

POST /catways/:id/reservations Create a reservation ▼

Books a berth for a given period. The API first verifies the catway exists, then checks for date conflicts before persisting the document.

Path Parameters

ParameterTypeRequiredDescription
id number required The catwayNumber of the parent catway.

Request Body

FieldTypeRequiredDescription
clientName string required Full name of the client.
boatName string required Name of the vessel.
startDate date required ISO 8601 start date string.
endDate date required ISO 8601 end date string. Must be after startDate.
Request example
JSON Body
{
  "clientName": "Jean Dupont",
  "boatName":   "Le Vent du Large",
  "startDate":  "2024-07-15",
  "endDate":    "2024-07-22"
}
● 201 Created
JSON
{
  "_id":          "665b2a4d3e5c6b001d9e0f23",
  "catwayNumber": 5,
  "clientName":  "Jean Dupont",
  "boatName":    "Le Vent du Large",
  "startDate":   "2024-07-15T00:00:00.000Z",
  "endDate":     "2024-07-22T00:00:00.000Z",
  "createdAt":   "2024-06-01T09:12:00.000Z",
  "updatedAt":   "2024-06-01T09:12:00.000Z"
}
● 400 Date Conflict

The requested period overlaps with an existing reservation on the same catway.

JSON
{ "message": "Catway already reserved for this period" }
● 404 Not Found

The catway identified by :id does not exist.

JSON
{ "message": "Catway not found" }
● 500 Internal Server Error
JSON
{
  "message": "Error creating reservation",
  "error":   "..."
}
⚠️
Real-world note: The conflict check uses an inclusive date range comparison — a reservation starting the same day another ends is considered a conflict. Plan for at least one day gap between consecutive bookings if your marina requires a turnover window.
GET /catways/:id/reservations List all reservations for a catway ▼

Returns all reservation documents for the given catway. Returns an empty array [] if none exist — no 404 is thrown for an empty list.

Path Parameters

ParameterTypeRequiredDescription
id number required The catwayNumber of the parent catway.
Request example
HTTP
GET /catways/5/reservations
● 200 OK
JSON
[
  {
    "_id":          "665b2a4d3e5c6b001d9e0f23",
    "catwayNumber": 5,
    "clientName":  "Jean Dupont",
    "boatName":    "Le Vent du Large",
    "startDate":   "2024-07-15T00:00:00.000Z",
    "endDate":     "2024-07-22T00:00:00.000Z",
    "createdAt":   "2024-06-01T09:12:00.000Z",
    "updatedAt":   "2024-06-01T09:12:00.000Z"
  },
  {
    "_id":          "665b2a4d3e5c6b001d9e0f24",
    "catwayNumber": 5,
    "clientName":  "Marie Martin",
    "boatName":    "Albatros III",
    "startDate":   "2024-08-01T00:00:00.000Z",
    "endDate":     "2024-08-10T00:00:00.000Z",
    "createdAt":   "2024-06-05T14:00:00.000Z",
    "updatedAt":   "2024-06-05T14:00:00.000Z"
  }
]
● 500 Internal Server Error
JSON
{
  "message": "Error fetching reservations",
  "error":   "..."
}
💡
Tip: An empty array [] is a valid 200 response — it simply means no reservations exist for that catway yet. This is different from the Catways list endpoint which returns 404 when empty.
GET /catways/:id/reservations/:idReservation Get a single reservation ▼

Returns one reservation identified by its MongoDB _id, scoped to the given catway.

Path Parameters

ParameterTypeRequiredDescription
id number required The catwayNumber of the parent catway.
idReservation ObjectId required The MongoDB _id of the reservation.
Request example
HTTP
GET /catways/5/reservations/665b2a4d3e5c6b001d9e0f23
● 200 OK
JSON
{
  "_id":          "665b2a4d3e5c6b001d9e0f23",
  "catwayNumber": 5,
  "clientName":  "Jean Dupont",
  "boatName":    "Le Vent du Large",
  "startDate":   "2024-07-15T00:00:00.000Z",
  "endDate":     "2024-07-22T00:00:00.000Z",
  "createdAt":   "2024-06-01T09:12:00.000Z",
  "updatedAt":   "2024-06-01T09:12:00.000Z"
}
● 404 Not Found

Returned if the reservation does not exist or belongs to a different catway.

JSON
{ "message": "Reservation not found" }
● 500 Internal Server Error
JSON
{
  "message": "Error fetching reservation",
  "error":   "..."
}
ℹ️
The query matches on both _id and catwayNumber. A valid reservation _id from a different catway will return 404 — this is intentional scoping behaviour.
PUT /catways/:id/reservations/:idReservation Update a reservation ▼

Updates any field of a reservation. The entire request body is spread onto the document. Send only the fields you want to change.

Path Parameters

ParameterTypeRequiredDescription
id number required The catwayNumber of the parent catway.
idReservation ObjectId required The MongoDB _id of the reservation to update.

Request Body

FieldTypeRequiredDescription
clientNamestringoptionalUpdated client name.
boatNamestringoptionalUpdated vessel name.
startDatedateoptionalNew start date. No conflict check is run.
endDatedateoptionalNew end date. No conflict check is run.
Request example
JSON Body
{
  "endDate": "2024-07-25"
}
● 200 OK

Returns the updated reservation document.

JSON
{
  "_id":          "665b2a4d3e5c6b001d9e0f23",
  "catwayNumber": 5,
  "clientName":  "Jean Dupont",
  "boatName":    "Le Vent du Large",
  "startDate":   "2024-07-15T00:00:00.000Z",
  "endDate":     "2024-07-25T00:00:00.000Z",
  "createdAt":   "2024-06-01T09:12:00.000Z",
  "updatedAt":   "2024-06-03T11:30:00.000Z"
}
● 404 Not Found
JSON
{ "message": "Reservation not found" }
● 500 Internal Server Error
JSON
{
  "message": "Error updating reservation",
  "error":   "..."
}
🚨
Real-world note: The update endpoint does not re-run the date conflict check. If you extend an endDate or shift a startDate, you must verify availability manually by calling GET /catways/:id/reservations first and checking for overlaps client-side.
