Catways

Manage marina berths (catways) — create, list, retrieve, update, and delete individual berth slots.

Base URL /catways

Data Model

Every catway is stored as a MongoDB document. Below is the full schema with field types, constraints, and descriptions.

Catway MongoDB / Mongoose
Field Type Constraints Description
_id ObjectId auto MongoDB unique identifier, generated automatically.
catwayNumber Number required unique Logical berth number within the marina. Must be unique across all catways.
catwayType String required enum Physical size of the berth.
short long
catwayState String required Free-text description of the current physical condition of the berth (e.g. "Good condition", "Rope broken").
createdAt Date auto Timestamp of document creation. Added automatically by Mongoose timestamps: true.
updatedAt Date auto Timestamp of last modification. Updated automatically by Mongoose.
ℹ️
Timestamps — Both createdAt and updatedAt are managed by Mongoose and are always present in responses. You never need to send them in a request body.

Validation Rules

  • catwayNumber must be a positive integer and must not already exist in the database. Attempting to create a catway with an already-used number returns 500.
  • catwayType accepts only short or long. Any other value is rejected with 400.
  • catwayState is a required free-text field. Sending an empty string or omitting it causes a validation error.
  • All three fields (catwayNumber, catwayType, catwayState) are mandatory on creation.
  • On update, only catwayType and catwayState can be changed. catwayNumber is immutable after creation.
  • Omitting all optional update fields results in a no-op (the document is not modified, but a 200 is returned).

Endpoints

POST /catways Create a new catway ▼

Creates a new marina berth. All three body fields are required. The catwayNumber must be unique.

Request Body

FieldTypeRequiredDescription
catwayNumber number required Unique integer identifying this berth.
catwayType string required Must be short or long.
catwayState string required Free-text condition description.
Request example
JSON Body 
{
  "catwayNumber": 12,
  "catwayType":   "long",
  "catwayState":  "Good condition, recently inspected"
}
● 201 Created
JSON
{
  "_id":          "664a1f3c2e4b5a001c8d9f12",
  "catwayNumber": 12,
  "catwayType":   "long",
  "catwayState":  "Good condition, recently inspected",
  "createdAt":    "2024-05-19T10:23:01.000Z",
  "updatedAt":    "2024-05-19T10:23:01.000Z"
}
● 400 Bad Request

Returned when a required field is missing or a value fails Mongoose validation (e.g. invalid enum).

JSON
{
  "issue":   "Bad request",
  "details": "catwayType: `medium` is not a valid enum value for path `catwayType`."
}
● 500 Conflict

Returned when catwayNumber already exists in the database.

JSON
{
  "issue":   "No processible action",
  "details": "This number of catway already exists. The actual last number is 15, so you can create number from 16"
}
⚠️
Real-world note: Even though the 500 response hints at the "next available" number, you should always query GET /catways first to understand the current state of berths before assigning a new number. Numbers are business-relevant and should not be assigned blindly.
GET /catways List all catways ▼

Returns the full array of catway documents. No query parameters are accepted.

Request example
HTTP
GET /catways
● 200 OK
JSON
[
  {
    "_id":          "664a1f3c2e4b5a001c8d9f12",
    "catwayNumber": 1,
    "catwayType":   "short",
    "catwayState":  "Good condition",
    "createdAt":    "2024-05-10T08:00:00.000Z",
    "updatedAt":    "2024-05-10T08:00:00.000Z"
  },
  {
    "_id":          "664a1f3c2e4b5a001c8d9f13",
    "catwayNumber": 2,
    "catwayType":   "long",
    "catwayState":  "Rope damaged on port side",
    "createdAt":    "2024-05-12T14:30:00.000Z",
    "updatedAt":    "2024-05-14T09:10:00.000Z"

Returned when the catways collection is empty (no documents at all).

JSON
{
  "issue":   "Database error",
  "details": "No catways found"
}

JSON
{
  "message": "Internal server error",
  "error":   "MongoNetworkError: ..."
}
GET /catways/:catwayNumber Get a single catway ▼

Returns one catway document identified by its numeric berth number.

Path Parameters

ParameterTypeRequiredDescription
catwayNumber number required The unique integer berth number.
Request example
HTTP
GET /catways/12
● 200 OK
JSON
{
  "_id":          "664a1f3c2e4b5a001c8d9f12",
  "catwayNumber": 12,
  "catwayType":   "long",
  "catwayState":  "Good condition, recently inspected",
  "createdAt":    "2024-05-19T10:23:01.000Z",
  "updatedAt":    "2024-05-19T10:23:01.000Z"
}
● 404 Not Found
JSON
{
  "issue":   "Database error",
  "details": "Catway not found"
}
● 500 Internal Server Error
JSON
{
  "message": "Internal server error",
  "error":   "..."
}
ℹ️
Note: The route uses :catwayNumber (a sequential integer), not the MongoDB _id. Always use the human-readable berth number in your URLs.
PUT /catways/:catwayNumber Update a catway ▼

Partially updates a catway. Only catwayType and catwayState can be changed — catwayNumber is immutable. Send only the fields you want to change.

Path Parameters

ParameterTypeRequiredDescription
catwayNumber number required Berth number of the catway to update.

Request Body

FieldTypeRequiredDescription
catwayType string optional short or long.
catwayState string optional New condition description.
Request example
JSON Body
{
  "catwayState": "Port-side rope replaced, now in excellent condition"
}
● 200 OK

Returns the updated document (uses returnDocument: 'after').

JSON
{
  "_id":          "664a1f3c2e4b5a001c8d9f12",
  "catwayNumber": 12,
  "catwayType":   "long",
  "catwayState":  "Port-side rope replaced, now in excellent condition",
  "createdAt":    "2024-05-19T10:23:01.000Z",
  "updatedAt":    "2024-05-20T08:45:00.000Z"
}
● 404 Not Found
JSON
{ "message": "Catway not found" }
● 500 Internal Server Error
JSON
{
  "message": "Error updating catway",
  "error":   "..."
}
💡
Tip: The update is non-destructive — omitting a field means it won't be changed. You can update only the state without touching the type, and vice versa.
DELETE /catways/:catwayNumber Delete a catway ▼

Permanently removes a catway. This action is irreversible. Ensure no active reservations exist for this berth before deleting.

Path Parameters

ParameterTypeRequiredDescription
catwayNumber number required Berth number of the catway to delete.
Request example
HTTP
DELETE /catways/12
● 200 OK

Returns a confirmation message and the deleted document.

JSON
{
  "message": "Catway deleted",
  "catway": {
    "_id":          "664a1f3c2e4b5a001c8d9f12",
    "catwayNumber": 12,
    "catwayType":   "long",
    "catwayState":  "Good condition, recently inspected",
    "createdAt":    "2024-05-19T10:23:01.000Z",
    "updatedAt":    "2024-05-19T10:23:01.000Z"
  }
}
● 404 Not Found
JSON
{ "message": "Catway not found" }
● 500 Internal Server Error
JSON
{
  "message": "Error deleting catway",
  "error":   "..."
}
🚨
Real-world note: The API does not currently cascade-delete reservations linked to this catway. Before deleting a berth, manually verify — or programmatically check via GET /catways/:id/reservations — that no active or future reservations exist for this slot. See the Reservations docs for details.
